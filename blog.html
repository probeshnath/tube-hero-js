Question 1 : Discuss the scope of var, let, and const

We can define or declare variable different way in JavaScript. Like Var,Let,Const. Let's discuss the scope of each:

Var

when Variables declared with var in function-scoped, it's means it's accessible only within the function. We can console log or return before initialition, we will get undefine. If a var variable is declared outside of any function or block, it becomes a global variable and is accessible throughout the entire script. var does not have block-level scope. Variables declared with var inside a block are still accessible outside of that block.

Let

When variables declared with let that is block-scoped, it's means that are accessible only within the block. When variables are hoisted to the top of their containing block but that are not initialized. We will get error. We Cannot access before initialized. Whan variables declared with let are not automatically added to the global scope. They are limited to the block or function.

Const

let , const variables are block-scoped, it's means that are accessible only within the block. When variables declared with const cannot be reassigned after they are initialized. However, the content of objects and arrays declared with const can be modified.

We Should use let and const for variable declarations due to their more predictable scoping and behavior. let should be used when the variable's value may change, while const should be used when the value should remain constant. Var should generally be avoided in favor of let and const to prevent unexpected behavior and scope-related issues.

Question 2 :Tell us the use cases of null and undefined

null and undefined are two special values in JavaScript that are often used to represent the absence of a meaningful value, but it's use in different contexts. There are the few common use cases of null and undefined:

We can explicitly assign null to a variable to indicate that it intentionally has no value and it is empty. null can be used to clear the value of a variable or property. Sometimes in functions, returning null can signify that no meaningful result is available.

undefined typically represents the absence of an assigned value. Variables that are declared but not initialized, or object properties that are not defined, then we will get an undefined value. When we don't pass any value in a function parameter and we called this function, then it becomes undefined. When we want to access an array element with an index that doesn't exist, Then we will get undefined.

Question 3 : What do you mean by REST API?

A REST API is a set of rules for making web services. It uses common actions like GET (get data), POST (create data), PUT (update data), and DELETE (delete data) to interact with resources, which can be anything like data or services. Each resource has a unique address (URI) to access it. REST APIs are stateless, meaning they don't remember past requests. Resources can have different formats, like JSON or XML. It also designed to be safe and repeatable. REST allows for layers in the system, like proxies, and follows a client-server structure. It's widely used in web and mobile apps to communicate with remote servers because it's simple, scalable, and works well with HTTP, making it a top choice for creating APIs.
